# STAT 184 Homework 4.3 and 4.4 Repository

## What this repository is for:
This repository contains my work for STAT 184 Homework 4.3
The main purpose of this repo is to show a reproducible data analysis project using Quarto, while also keeping all of the files needed for reproducibility in one public GitHub repository.

## Data:
This project uses multiple sources of data:
- scraped data from a publicly available busiest airport passenger traffic page on Wikipedia, used for the airport analysis section in HW 4.3
            The airport data were collected from an online public source and then cleaned and organized in R so they could be used for comparison across airports and years
- simulated or generated data from the Monte Carlo numeric integration section
            The Monte Carlo section does not have a separate raw data file in the repository because that section depends on simulated points generated directly in code during the analysis
- `calcium.csv` for the calcium treatment data used in the assignment’s wrangling, plotting, and prompting section
             The calcium dataset is included in the repo because it is one of the direct files used in the assignment

## Current plan:
My current plan is to keep all files for the assignment organized in one public repository, including the source QMD file, the rendered PDF, the dataset, and any other files needed to support reproducibility. I also want this repository to clearly show the GitHub workflow skills learned in class.

More specifically, I want this repo to demonstrate:
- how to set up and maintain a public GitHub repository
- how to keep assignment files organized and clearly named
- how to use branches instead of doing all work directly on the main branch
- how to make meaningful commits that reflect actual changes in the project
- how to create and resolve issues as part of tracking work
- how to open and merge pull requests
- how to keep the main branch updated with the final version of the project

I also want the repo to be easy for someone else to open and understand.
A person visiting this repository should be able to see what the project is about, what files belong to the assignment, where the data came from, and how the work was managed through GitHub.
